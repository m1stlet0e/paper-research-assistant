## 2️⃣ 飞书多维表格

### 功能

- 自动创建论文数据表格
- 存储论文标题、作者、发表时间、研究方法、数据集等
- 支持筛选、统计、多视图

### 字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 论文标题 | 文本 | 论文标题 |
| 作者 | 文本 | 作者列表 |
| 发表时间 | 日期 | 发表日期 |
| 研究方法 | 文本 | 使用的研究方法 |
| 数据集 | 文本 | 使用的数据集 |
| 核心结论 | 文本 | 主要结论 |
| PDF链接 | URL | 论文下载链接 |
| 研究课题 | 文本 | 关联的研究课题 |
| 检索时间 | 日期 | 检索日期 |

### API调用

```bash
# 创建表格
curl -X POST http://localhost:6000/api/bitable \
  -H "Content-Type: application/json" \
  -d '{"action": "create", "name": "我的论文库"}'

# 添加论文
curl -X POST http://localhost:6000/api/bitable \
  -H "Content-Type: application/json" \
  -d '{"action": "add", "paper": {...}, "topic": "LLM"}'
```

## 3️⃣ 飞书知识库集成

### 功能

- 自动创建知识库空间
- 研究报告归档
- 按课题分类管理
- 自动生成目录结构

### 归档内容

1. **研究报告** - Markdown格式的完整报告
2. **论文列表** - 所有检索到的论文
3. **可视化文件** - 知识图谱、对比矩阵等

### API调用

```bash
# 创建知识库
curl -X POST http://localhost:6000/api/wiki \
  -H "Content-Type: application/json" \
  -d '{"action": "create", "name": "AI论文预研知识库"}'

# 归档研究
curl -X POST http://localhost:6000/api/wiki \
  -H "Content-Type: application/json" \
  -d '{"action": "archive", "topic": "LLM", "report": "...", "papers": [...]}'
```

## 4️⃣ 每日论文推送

### 功能

- 每天定时推送最新论文
- 支持关键词订阅
- 自动过滤和排序

### 配置

在 `config/feishu_config.py` 中配置：

```python
DAILY_PUSH_CONFIG = {
    "enabled": True,
    "time": "08:00",  # 每天8点推送
    "keywords": ["LLM", "Transformer", "AI", "机器学习"],
    "max_papers": 10,  # 每天最多推送10篇
}
```

### 手动触发

```bash
curl -X POST http://localhost:6000/api/push
```

## 🔧 配置飞书机器人

### 步骤1：创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 创建企业自建应用
3. 记录 App ID 和 App Secret

### 步骤2：配置事件订阅

1. 进入应用 → 事件订阅
2. 配置请求网址：`http://你的服务器地址:6000/webhook`
3. 订阅事件：`im.message.receive_v1`

### 步骤3：配置权限

在应用 → 权限管理 中添加：

- `im:message` - 获取与发送消息
- `bitable:app` - 多维表格操作
- `wiki:wiki` - 知识库操作
- `docs:doc` - 文档操作

### 步骤4：发布应用

1. 配置完成后发布应用
2. 在飞书中搜索应用名称
3. 发送消息测试

## ❓ 常见问题

### Q: Webhook验证失败？

确保：
1. 服务器地址可以从公网访问
2. 返回了正确的 challenge 响应
3. 使用 HTTPS（生产环境）

### Q: 消息没有响应？

检查：
1. 是否订阅了 `im.message.receive_v1` 事件
2. 是否有 `im:message` 权限
3. 查看服务器日志

### Q: 每日推送不工作？

检查：
1. `DAILY_PUSH_CONFIG.enabled` 是否为 True
2. 服务器是否在运行
3. 关键词列表是否为空

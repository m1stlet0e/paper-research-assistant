# AI论文全自动预研助手 - 项目说明书

## 📋 项目概述

**项目名称：** AI论文全自动预研助手

**赛道：** 学术龙虾 (Academic Lobster)

**一句话介绍：** 基于OpenClaw的多Agent协作系统，实现从课题输入到文献综述生成的全流程自动化

---

## 🎯 核心问题

### 痛点分析

**1. 科研人员/研究生在开题阶段需要阅读海量文献，耗时极长且容易遗漏关键信息**

- 传统方式：手动浏览arXiv、Google Scholar、知网等平台
- 问题：
  - 信息过载：每天新发论文数千篇
  - 时间成本：完整调研需要1-2周
  - 容易遗漏：手动检索容易错过重要论文
  - 效率低下：重复劳动多，缺乏系统性

**2. 手动整理文献元数据非常枯燥，效率低下**

- 传统方式：手动记录论文标题、作者、摘要、结论
- 问题：
  - 工作量大：每篇论文需要5-10分钟整理
  - 易出错：手动输入容易出错
  - 难维护：缺乏结构化存储，难以检索
  - 无法追踪：不知道哪些论文已经读过

---

## 💡 解决方案

### 核心功能

#### 1. 智能文献检索与筛选

**功能描述：**
- 用户输入一个课题（如"大模型在K12英语口语教学中的应用"）
- 龙虾自动去arXiv、Google Scholar、知网搜索近3年的高引用论文
- 智能去重和相关性排序
- 生成论文元数据表格（Excel格式）

**技术实现：**
- **多数据源检索：** 主Agent分析课题关键词，调用arXiv API、Google Scholar API、知网API
- **智能去重：** 基于论文标题、DOI、作者等信息去重
- **相关性排序：** 基于关键词匹配度和引用次数排序
- **结构化输出：** 生成Excel表格，包含论文元数据

**创新点：**
- 多源检索，不遗漏重要论文
- 智能去重，避免重复检索
- 相关性排序，优先展示重要论文

---

#### 2. 深度阅读与信息提取

**功能描述：**
- 自动阅读PDF，提取核心观点、实验数据、所用模型、结论
- 生成一个Excel对比表格
- 提取论文的关键点、局限性等

**技术实现：**
- **PDF解析：** 使用PyPDF2解析PDF文件
- **信息提取：** 使用LLM提取论文的核心信息
  - 标题、作者、摘要、关键词
  - 实验方法、数据集、模型架构
  - 核心结论、创新点、局限性
- **结构化存储：** 生成Excel表格，方便对比

**创新点：**
- 自动提取，无需人工阅读
- 结构化存储，便于后续分析
- 提取关键点和局限性，提供全面视角

---

#### 3. 自动生成综述草稿

**功能描述：**
- 基于提取的内容，自动撰写一份带引用的文献综述初稿（Markdown格式）
- 生成文献地图（思维导图）
- 支持自定义综述结构和内容

**技术实现：**
- **综述结构：** 自动生成研究背景、主要发现、研究趋势、总结展望
- **内容生成：** 使用LLM基于提取的论文信息生成综述内容
- **引用格式：** 自动添加正确的引用格式
- **思维导图：** 生成文献地图，展示研究领域的关系

**创新点：**
- 自动生成，节省大量时间
- 结构清晰，逻辑完整
- 支持自定义，灵活性高

---

#### 4. 每日追踪与推送

**功能描述：**
- 每天自动监测该领域新发的Paper
- 如果有突破性进展，自动推送到用户的邮箱或飞书
- 记录到知识库，避免重复推送

**技术实现：**
- **定时任务：** 使用cron每天早上8点自动执行
- **增量检索：** 只检索新发的论文
- **智能判断：** 基于论文标题、摘要判断是否有突破性进展
- **推送通知：** 推送到Feishu，实时提醒

**创新点：**
- 持续更新，保持知识最新
- 智能判断，只推送重要信息
- 避免重复，提高效率

## ✨ 核心亮点

### 1. 真正的自动化

- **全流程自动化：** 从课题输入到报告生成，无需人工干预
- **智能异常处理：** 自动处理PDF解析失败、API请求失败等情况
- **支持中断恢复：** WAL系统记录所有状态，支持从中断处恢复

**示例：**
```python
# 自动处理异常
try:
    result = agent.execute()
except Exception as e:
    # 自动记录错误
    memory.log("error", {"error": str(e)})
    # 自动重试
    result = agent.execute()
```

### 2. 多Agent智能协作

- **主Agent负责整体规划：** 分析课题、规划任务、分配资源
- **子Agent并行处理：** ArxivSearcher、PDFParser、ReviewWriter等并行执行
- **智能任务分配：** 根据任务优先级和依赖关系智能调度

**架构优势：**
- 模块化设计，易于扩展
- 并行处理，提高效率
